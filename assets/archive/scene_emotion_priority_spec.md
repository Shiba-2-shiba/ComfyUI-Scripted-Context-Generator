# シーン描写・感情重視 改善仕様書

## 1. 文書の目的

本仕様書は、現行の `ComfyUI-Scripted-Context-Generator` を以下の方向へ改善するための実装計画を定義する。

- **日常系を多めに維持する**
- **プロンプトを品質語ではなく場面描写に使う**
- **感情を重要な要素として扱う**
- **不自然なエフェクト、異物、ゴミっぽい混入を抑える**
- **アニメモデル向けの自然言語プロンプトとして使いやすくする**

本書は「どの順で、何を、どの基準で直すか」を決めるための仕様書であり、実装コードそのものではない。

---

## 2. 背景認識

現状の課題は次の 4 点に整理する。

1. 日常系ロケーションが多く、体感上は同じような場所に見えやすい
2. シーン描写よりも、雰囲気語や画作り語にプロンプト枠を使っている箇所がある
3. 感情表現が「付け足し」に寄りやすく、行動・姿勢・視線・状況と十分に結び付いていない
4. 一部の FX 語彙、異物語彙、データ上のノイズが生成結果の不自然さを生みやすい

ただし、**日常系が多いこと自体は不具合ではなく意図された方向性**とみなす。よって本改善では、日常系比率を下げることではなく、**日常系の中でシーン差・感情差を大きくすること**を主目的とする。

---

## 3. 設計方針

### 3.1 基本方針

- **日常系優勢は維持する**
- **シーン描写を主軸にする**
- **感情を主軸にする**
- **品質系・レンダリング系の語を既定では弱める**
- **アニメモデルで効く、具体的で破綻しにくい文に寄せる**

### 3.2 プロンプト予算の優先順位

最終プロンプトで優先する情報量の順序は以下とする。

1. 主人公の状況
2. 場所と場面の具体
3. 感情状態とその表出
4. 行動・姿勢・視線・手の動き
5. 場面に必要な小物
6. 雰囲気補助
7. スタイル語・品質語

### 3.3 感情の位置付け

感情は garnish 的な付加要素ではなく、以下を決めるための**重要な生成軸**とする。

- 行動の選び方
- 立ち方、座り方、視線、口元、手元
- 場面内の距離感
- 時間帯や空気感の相性
- テンプレートの文構造

---

## 4. 非目標

以下は今回の改善対象外とする。

- 高品質化プロンプトの拡充
- フォトリアル向けカメラ語彙の強化
- NSFW 方向への最適化
- LLM 導入前提の再設計

---

## 5. 目標状態

改善後は、同じ「日常系」でも次のような差が自然に出る状態を目指す。

- 同じカフェでも
  - 待ち合わせ前で落ち着かない
  - 読書に没頭して静か
  - 仕事の合間で少し疲れている
  - 雨宿りでほっとしている
- 同じ通学・通勤系でも
  - 急いでいる
  - ぼんやりしている
  - 期待している
  - 気まずさがある

つまり、**大分類としては日常でも、シーンと感情の掛け算で体感差を出す**。

---

## 6. 優先度順 実装計画

## P0. 生成方針の明文化と既定値の整理

### 目的

後続実装で判断がぶれないよう、「何を優先して生成するか」を明示する。

### 変更方針

- 既定モードを「シーン描写・感情重視」に寄せる
- `meta_style` は任意要素に格下げする
- `meta_mood` は抽象的な atmosphere 語ではなく、場面と人物に効く情報へ寄せる
- 日常系の比率目標を仕様化する

### 要件

- デフォルト出力では品質語より場面語が多いこと
- 感情は必ず人物側の表出に落ちること
- 日常系比率は保持しつつ、同一 broad scene の反復感を下げること

### 受け入れ基準

- 仕様書に沿った出力モード名を定義する
- 既定テンプレートで `meta_style` が必須でなくなる
- 評価時に「場面描写語数 > スタイル語数」となる

---

## P1. 感情モデルを garnish から主軸へ昇格

### 目的

感情を「最後に足す語」ではなく、行動・姿勢・表情・状況を決める中心軸にする。

### 変更対象

- `mood_map.json`
- `vocab/garnish/logic.py`
- `nodes_garnish.py`
- 行動プールとテンプレート接続部

### 新ルール

- 感情は以下の 4 層に分けて扱う
  - `emotion_core`: 喜び、緊張、安堵、寂しさ、期待、気まずさなど
  - `emotion_intensity`: 弱、中、強
  - `emotion_expression`: 表情・視線・口元
  - `emotion_behavior`: 手元、姿勢、微動作
- 最終文では、最低 1 つ以上の **身体的な表出** を含める
- 感情と矛盾する表現は出さない
- 感情に応じて action 候補も分岐させる

### 実装要件

- 感情ごとに「表情」「視線」「姿勢」「手元」「行動傾向」の対応表を持つ
- `sample_garnish` は effect 優先ではなく expression / behavior 優先で構成する
- 感情が action や loc の文脈と矛盾した場合は補正する
- `_is_out_of_context` を実装し、場違い表現を実際に落とす

### 受け入れ基準

- サンプル監査で、最終文の 85% 以上に感情の身体的表出が入る
- 抽象的 mood 名だけで終わる文を既定モードで禁止する
- 感情語が effect 語より多く出るのではなく、**感情に基づく行動・表情**が増える

---

## P2. 日常系を維持したまま、場面差を増やす

### 目的

「日常系が多い」という意図を壊さずに、同じ場所に見える問題を減らす。

### 変更対象

- `scene_compatibility.json`
- `background_packs.json`
- `action_pools.json`
- `nodes_scene_variator.py`
- `nodes_dictionary_expand.py` の `ThemeLocationExpander`

### 新ルール

- 日常系は「学校」「通勤通学」「買い物」「カフェ」「自室」「近所」「雨天」「帰り道」などの**生活フェーズ**に分ける
- ロケーション差だけでなく、以下の scene axis を強化する
  - `purpose`
  - `progress`
  - `social_distance`
  - `time`
  - `weather`
  - `obstacle`
- 同一 loc でも「状況違い」の action を増やす
- `universal_locs` は全削除ではなく、**日常系優先を維持しつつ粒度を再編**する

### 実装要件

- broad な日常ロケーションは維持してよい
- ただし各日常ロケーションに最低限、以下の差分を持たせる
  - 一人の時間
  - 誰かを意識する時間
  - 移動中
  - 休息中
  - 作業中
- action pool は意味距離を確保する
- object が固定化するプールは追加・分散する

### 目標値

- 日常系シェアは既定で 60% から 75% を許容範囲とする
- ただし top broad scene の反復感は下げる
- 特定 loc における単一 object の conditional rate を 0.55 未満に抑える

### 受け入れ基準

- 日常系比率を維持したまま、ロケーション別 object 偏りが改善する
- broad scene は日常中心でも、同 seed 群で状況差が視認できる
- unused loc が減るか、意図的未使用として整理される

---

## P3. シーン語を増やし、画作り語を既定で減らす

### 目的

アニメ生成向けに、場面そのものの描写を太くし、品質系・レンダリング系の語がプロンプト予算を食い過ぎないようにする。

### 変更対象

- `mood_map.json`
- `templates.txt`
- `vocab/templates_intro.txt`
- `vocab/templates_body.txt`
- `vocab/templates_end.txt`
- `nodes_simple_template.py`

### 新ルール

- `meta_style` は既定では省略可能とする
- `meta_mood` は atmosphere 名ではなく、場面と人物に効く文へ置き換える
- テンプレートは説明文調を弱め、視覚的な scene description を優先する

### 置換方針

- 置き換える対象
  - bloom
  - film grain
  - light leaks
  - ambient occlusion
  - volumetric light
  - prismatic light leaks
- 優先する対象
  - どこで
  - 何をしていて
  - どんな感情で
  - どう見えているか

### 受け入れ基準

- 既定テンプレートで style 語がなくても成立する
- mood 展開文にレンダリング用語が残っても、既定モードでは採用されない
- 人物と場面を読むだけで絵が想像できる文になっている

---

## P4. 不自然 FX・異物・ゴミ語彙の抑制

### 目的

不自然な粒子、ゴミ、意味上のノイズが画像に混入する原因を減らす。

### 変更対象

- `PromptCleaner`
- `background_packs.json`
- `action_pools.json`
- `garnish` vocab
- lint / test 系スクリプト

### 新ルール

- `imaginary` 表記を既定データから禁止する
- 日常系ロケーションでは trash / debris を原則禁止する
- grimy / backstreet / ruined 系の pack のみ、明示タグ付きで許可する
- FX denylist を「粒子系」だけでなく「画面加工系」に広げる

### 実装要件

- 語彙 lint を追加し、以下を検出する
  - imaginary
  - 不要な trash 語
  - 強すぎる effect 語
  - 品質系レンダリング語
- allowlist と denylist を pack type ごとに持てるようにする

### 受け入れ基準

- 日常系出力で trash / debris / imaginary が 0 件
- disallowed FX が 0 件
- gritty 系ロケーションだけは例外許可が働く

---

## P5. テンプレートを「説明」から「視覚化」へ寄せる

### 目的

文章の説明っぽさを減らし、絵として立ち上がる文にする。

### 新ルール

- 「The location is ...」型の説明文を減らす
- 状況、人物、感情を 1 本の scene sentence としてつなぐ
- mood ラベル名をそのまま出さない

### テンプレート要件

- 主語説明
- 状況説明
- 感情表出
- 背景の具体

上記 4 要素を含みつつ、ナレーション感を抑える。

### 受け入れ基準

- 既定テンプレート群で mood key の生出力を禁止
- 文頭・文末の定型反復率を下げる
- 同一要素でも複数の自然な言い回しがある

---

## P6. 監査と評価を仕様化

### 目的

改善が「好み」ではなく、観測可能な差分として判断できるようにする。

### 継続利用するもの

- `tools/run_bias_audit.py`
- `assets/test_fx_cleanup.py`
- `assets/validate_compatibility.py`

### 追加指標

- daily-life share
- broad scene entropy
- location ごとの object concentration
- emotion embodiment rate
- abstract style term rate
- unwanted noun rate

### 判定ゲート

- 日常系比率が目標範囲内
- unwanted noun rate = 0
- disallowed FX rate = 0
- emotion embodiment rate >= 0.85
- broad scene の top1 share が現状より悪化しない
- location ごとの object concentration が改善する

---

## 7. 実装順序

実装順は以下を推奨する。

1. P0 生成方針の固定
2. P1 感情モデルの再設計
3. P4 ノイズ語彙の除去
4. P3 シーン語優先の mood / template 再編
5. P2 日常系シーン差の拡張
6. P5 テンプレートの自然文再設計
7. P6 監査指標の追加

理由は、**感情とノイズの扱いを先に決めないと、シーン強化の方向性がぶれるため**である。

---

## 8. 推奨マイルストーン

### M1

- 方針固定
- 感情モデル定義
- 不要語彙 blacklist 確定

### M2

- mood / garnish / template の再編
- 既定モードの scene-first 化

### M3

- 日常系 location / action の差分拡張
- object 偏り対策

### M4

- 監査自動化
- baseline 更新
- リリース判定

---

## 9. 最終的に目指す出力の性質

最終的な出力は、次の条件を満たすことを目標とする。

- 日常系が中心でも、同じ絵に見えない
- 感情がラベルではなく表情・姿勢・行動に出る
- 場面が先に立ち、スタイル語は従属する
- アニメモデルで扱いやすい
- ノイズ的な異物が混じりにくい

以上を、本プロジェクトの既定方針とする。
