# Compaction Policy (Common)

## セッション終了のトリガー

以下のいずれかが発生したとき、セッション終了とみなして圧縮を実施する：

- ユーザーが明示的に「終了」「セッション終わり」と宣言したとき
- タスクの主要成果物（ファイル・出力）がすべて完成したとき
- 会話が自然に一区切りついたとき（次のリクエストが明確でない）

---

## session_notes.md の圧縮ルール

セッション終了時に `session_notes.md` を以下の形式に削減する（最大10箇条）：

```
## [YYYY-MM-DD] セッション要約

### 新事実
- （このセッションで判明した重要な情報）

### 決定事項
- （このセッションで行った重要な判断）

### 未解決
- （次セッションに持ち越す問題・疑問）

### 次アクション
- （次にやること）
```

---

## global_notes.md への昇格基準

`session_notes.md` の内容を `global_notes.md` に移すのは：

- **昇格すべきもの**: プロジェクト全体に永続的に影響する事実・制約・決定
  - 例: 「このプロジェクトは Python 3.11 限定」「本文の一人称は『俺』」
- **昇格しないもの**: セッション固有の一時情報・やり取りの詳細・完了したタスク

---

## ログへの記録

圧縮時に合わせて実施：

- 重要な判断 → `decisions_log.md` に追記
- ファイル変更 → `change_log.md` に追記
